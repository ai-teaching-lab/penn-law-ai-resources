Faculty Orientation

Understanding Agentic AI

A brief orientation for faculty on agentic AI versus chatbot tools — what changes, why it matters, and when to reach for which.

Last updated May 14, 2026
Already familiar with the agentic-vs-chat distinction? Skip ahead to the safety guide for faculty use of agentic AI tools.

The short version

Chat AI answers a question. You type, it responds, and that’s the transaction. Each exchange is self-contained.

Agentic AI does a task. You give it a goal, and it works toward that goal across multiple steps — reading files, running programs, searching the web, sending messages, modifying documents — until the task is done.

The line is blurring. Modern Claude and ChatGPT can do some agentic things from within a chat (search the web, run a small calculation, read a file you uploaded). But “agentic AI” usually refers to dedicated tools — Anthropic’s Claude Code, OpenAI’s Codex CLI, Google’s Gemini CLI, Anthropic’s Claude Cowork, and their analogs — purpose-built to take sustained, autonomous action on your computer or in your accounts.

A useful analogy: Chat AI is like a research assistant who answers questions when you walk over to their desk.

Agentic AI is like a research assistant you hand a project to, with keys to your office and your filing cabinet, while you step out for the afternoon.

Both are useful. Both have appropriate uses. But the second one requires you to think carefully — beforehand — about which keys you hand over.

What changes when AI becomes agentic

  1. It reads on its own. A chat tool reads what you paste. An agent reads what it finds — files in a folder you point it at, messages in a connected inbox, web pages it decides to visit. You set the scope; it picks what to look at within that scope.
  2. It acts; it doesn’t just describe. A chat tool will tell you the command to delete duplicate files. An agent will run the command. A chat tool will draft an email. An agent will send it.
  3. It plans across many steps. Agents work in a loop: form a plan, take an action, observe the result, revise the plan, take the next action. A single instruction from you might trigger dozens of intermediate steps.
  4. It uses “tools.” In AI parlance, a tool is anything the agent can call: read a file, run a terminal command, fetch a webpage, send an email, query a calendar. The tools available define what the agent can do. Adding a tool (like a Gmail connector) expands the agent’s reach.
  5. It works between your prompts. The agent may be working for a minute or for ten between your messages. It doesn’t pause to ask permission for every step unless you’ve configured it to.

Why the difference matters

  • Speed and scope. Agents can do in minutes what would take hours of clicking and typing. The flip side is that they can also do damage in minutes.
  • Reviewability. Chat outputs are text you can read. Agentic outputs are actions: files changed, messages sent, calendar invites accepted. Reviewing what an agent did is more involved than reviewing what a chatbot wrote.
  • Trust setup. With chat, you decide each turn what to share. With an agent, the agent decides what to read within whatever scope you’ve granted. That makes the initial setup — folder, permissions, connectors — far more consequential than what you paste into any single chat.
  • Reversibility of errors. Chat errors are wrong answers you can ignore. Agent errors are wrong actions — a deleted file, a misdirected email, an inappropriately scheduled task. Some are easy to undo; some are not.

When to reach for which

Chat AI suits drafts, summaries, explanations, brainstorming, and questions where the input fits in a message and the output is text you’ll review.

Agentic AI suits tasks that span many files, require chained steps, or involve making changes rather than describing them — particularly repetitive or well-defined work.

The current tools

Faculty considering agentic AI today are mostly looking at three rough groups of tools. All require a paid subscription; the safety setup matters more than the tool choice.

Coding and file agents (terminal-based)

The most mature category — and the most useful for general document and file work, despite the “coding” framing. Claude Code (Anthropic), OpenAI Codex, and Gemini CLI (Google) all run primarily in your terminal or IDE. They read repositories and folders, edit files, run shell commands, and iterate across many steps. Originally built for software development, increasingly used for any task that touches files: document workflows, data analysis, and course material processing. The most flexible category, but requires comfort with command-line tools. Of the three, Claude Code is the most polished for non-developer use; Codex is the closest OpenAI parallel; Gemini CLI is open-source (Apache 2.0).

App-based agents

A friendlier way to use agentic AI without a terminal. Claude Cowork (Anthropic) is the practical option today — a desktop or browser app where you hand the agent a task (research, drafting, analysis, summarization) and it works in the background and returns results. Less raw filesystem power than the terminal agents above, but no terminal required — closer to delegation than command-line interaction. Cowork’s marketing emphasizes connectors to email, calendar, and drive, but for now, Penn Law faculty cannot connect agentic AI tools (Cowork, Claude, ChatGPT, or any other) to Penn’s O365 instance — email, calendar, OneDrive, Teams. Treat Cowork as a delegation interface for self-contained tasks, not as a productivity-app integrator for Penn work. Microsoft has announced a similar product (Copilot Cowork) but it’s not yet generally available. Penn Carey Law doesn’t currently provide a managed account; individual subscription is required.

Agentic features in chat tools

Modern Claude and ChatGPT now include agentic capabilities inside their normal chat apps — browsing the web, running code, working with files you upload, and (in ChatGPT’s case) extended Agent Mode sessions. These are lighter-weight than the dedicated tools above but exhibit the same dynamics: the agent reads things, takes actions, and works between your prompts. The safety considerations are the same.

For product cards with links and installation guidance, see the Agentic AI tab of the AI Resources portal.

A note on terminology

“Agent,” “assistant,” “copilot,” and “AI tool” are used loosely and inconsistently across products. The distinction that matters isn’t the label. It’s whether the tool can take action on your behalf without each step being explicitly approved. If it can, treat it as agentic — and apply the cautions in the companion security document.
